Graph API Setup Guide — EDP 2026

For IT administrators · Microsoft Azure Active Directory · Estimated time: 15–20 minutes

What this does

Once configured, the EDP app will automatically send branded HTML emails from edp-noreply@landis-gyr.com whenever an employee submits a form, gets approved, or needs to correct their form — with zero clicks required from managers.

Without this setup, the app falls back to opening Outlook with a pre-filled email (still works, just requires a manual send).

1

Create the shared mailbox in Microsoft 365

This is the email address that will appear as the sender on all EDP notifications.

  1. Go to Microsoft 365 Admin Center → Teams & Groups → Shared Mailboxes
  2. Click + Add a shared mailbox
  3. Set the name: EDP Portal
  4. Set the email: edp-noreply@landis-gyr.com
  5. Click Save
💡 No licence needed — shared mailboxes are free in Microsoft 365.
2

Register an app in Azure Active Directory

  1. Go to portal.azure.com and sign in with your admin account
  2. Search for "Azure Active Directory" → click App registrations
  3. Click + New registration
  4. Name: EDP Portal Email
  5. Supported account types: Accounts in this organizational directory only
  6. Redirect URI: leave blank
  7. Click Register

After registering, you'll see the app overview page. Copy these two values — you'll need them later:

Application (client) ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx ← copy this Directory (tenant) ID: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx ← copy this
3

Add API permissions

  1. In your app registration, click API permissions in the left menu
  2. Click + Add a permission
  3. Choose Microsoft Graph
  4. Choose Application permissions (not Delegated)
  5. Search for Mail.Send and check it
  6. Click Add permissions
  7. Click Grant admin consent for Landis+Gyr → confirm Yes
⚠️ The "Grant admin consent" button requires a Global Administrator account. If you don't have one, ask your Azure AD admin.
✅ The permission status should show a green checkmark: Granted for Landis+Gyr
4

Create a client secret

  1. In your app registration, click Certificates & secrets
  2. Click + New client secret
  3. Description: EDP Portal Secret
  4. Expiry: choose 24 months
  5. Click Add
  6. Immediately copy the Value (it disappears after you leave the page!)
⚠️ Copy the secret VALUE now — it will be hidden after you navigate away. If you lose it, you'll need to create a new one.
Client secret value: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx ← copy this
5

Add the values to the EDP app

Open src/App.jsx in a text editor (e.g. Notepad) and find this section near the top:

const GRAPH_CONFIG = { clientId: "", // ← paste Application (client) ID here tenantId: "", // ← paste Directory (tenant) ID here clientSecret: "", // ← paste client secret Value here senderEmail: "edp-noreply@landis-gyr.com", };

Fill in the three values you copied, so it looks like:

const GRAPH_CONFIG = { clientId: "a1b2c3d4-e5f6-7890-abcd-ef1234567890", tenantId: "f0e1d2c3-b4a5-6789-abcd-ef0987654321", clientSecret: "abc~defghijklmnopqrstuvwxyz0123456789", senderEmail: "edp-noreply@landis-gyr.com", };

Save the file, then restart the app (run START-APP.bat again).

✅ Once configured, the manager dashboard header will show: "✅ Graph API active"
!

Security note

Storing the client secret directly in the app code is fine for internal/local use. If you later deploy the app publicly online, you should move the secret to a backend API (e.g. an Azure Function) so it's not visible in the browser. For now, since only you and your IT team can access the source files, this setup is acceptable.

⚠️ Never share your App.jsx file publicly or commit it to a public Git repository with the secret filled in.

Troubleshooting

  • "Insufficient privileges" → Admin consent was not granted in Step 3. Repeat step 3.
  • "Invalid client secret" → Secret may have expired or was copied incorrectly. Create a new one in Step 4.
  • Emails not arriving → Check that the shared mailbox was created correctly and that senderEmail matches exactly.
  • Still showing "Outlook fallback" → All three fields (clientId, tenantId, clientSecret) must be filled. Check for extra spaces.
