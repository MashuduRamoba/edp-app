SharePoint Setup Guide — EDP 2026

For IT administrators · Microsoft 365 SharePoint · Estimated time: 10 minutes · Requires: Azure App already set up (GRAPH-SETUP-GUIDE.html)

What this does

Once configured, all EDP data (employee forms, manager accounts, notifications) is stored in SharePoint Lists instead of the browser. This means:

  • Any employee on any computer can submit their form
  • Any manager on any computer can see all submissions instantly
  • Data is never lost if a browser is cleared or a computer replaced
  • HR can view and edit the raw data directly in SharePoint like an Excel table
1

Add SharePoint permission to your Azure App

You already set up an Azure App in GRAPH-SETUP-GUIDE.html. You just need to add one more permission.

  1. Go to portal.azure.com → Azure Active Directory → App registrations
  2. Click your EDP Portal Email app
  3. Click API permissions → + Add a permission
  4. Choose Microsoft Graph → Application permissions
  5. Search for Sites.ReadWrite.All and check it
  6. Click Add permissions
  7. Click Grant admin consent for Landis+Gyr → Yes
✅ You should now have two permissions with green checkmarks: Mail.Send and Sites.ReadWrite.All
2

Find your SharePoint site URL and path

You need two values from your SharePoint site:

siteUrl — the full URL
https://landis-gyr.sharepoint.com/sites/HR
sitePath — just the path part
/sites/HR

To find these:

  1. Open your SharePoint site in the browser
  2. Copy the URL from the address bar — e.g. https://landis-gyr.sharepoint.com/sites/HR
  3. The siteUrl is the full URL above
  4. The sitePath is everything after .com — e.g. /sites/HR
⚠️ If your SharePoint URL looks like https://landis-gyr.sharepoint.com/ (no path), use / as the sitePath.
3

Add the values to App.jsx

Open src/App.jsx in Notepad and find the GRAPH_CONFIG section near the top:

const GRAPH_CONFIG = { clientId: "your-client-id", // already filled in tenantId: "your-tenant-id", // already filled in clientSecret: "your-secret", // already filled in senderEmail: "edp-noreply@landis-gyr.com", // SharePoint config — ADD THESE TWO: siteUrl: "", // ← paste your full SharePoint site URL here sitePath: "", // ← paste the path portion here };

After filling in, it should look like:

const GRAPH_CONFIG = { clientId: "a1b2c3d4-...", tenantId: "f0e1d2c3-...", clientSecret: "abc~defg...", senderEmail: "edp-noreply@landis-gyr.com", siteUrl: "https://landis-gyr.sharepoint.com/sites/HR", sitePath: "/sites/HR", };

Save the file and restart the app (START-APP.bat).

4

Verify it's working

  1. Open the app and log in as a manager
  2. Go to the 🗄️ Database Admin tab
  3. You should see a 🟢 SharePoint Connected banner at the top
  4. The first time the app loads, it will automatically create 3 lists in SharePoint: EDP_Employees EDP_Managers EDP_Notifications
  5. Register a test employee — then open SharePoint and go to Site contents to confirm the EDP_Employees list was created with a row
✅ Once connected, all saves, submits, approvals and rejections write to SharePoint automatically in the background.
i

Viewing data directly in SharePoint

Since data is now stored as SharePoint Lists, HR can view and edit it directly:

  1. Go to your SharePoint site → Site contents
  2. You'll see EDP_Employees, EDP_Managers, EDP_Notifications
  3. Click EDP_Employees to see a table of all employees, their status, job title, submission dates etc.
  4. You can filter, sort, and export to Excel directly from SharePoint
⚠️ The formData column contains the full form as JSON text. Don't edit it directly in SharePoint — always use the EDP app to make changes.

Troubleshooting

  • Still shows 🟡 Local storage → Check that both siteUrl and sitePath are filled in, and that clientId, tenantId, clientSecret are all set.
  • "Access denied" errors in browser console (F12) → The Sites.ReadWrite.All permission was not granted admin consent. Repeat Step 1.
  • Lists not created automatically → Check that the Azure app has Sites.ReadWrite.All permission. Then reload the app as a manager.
  • Data shows in app but not in SharePoint → The site URL or path may be slightly wrong. Double-check Step 2.
  • Existing localStorage data not migrating → After connecting SharePoint, use the 🗄️ Database Admin → Export JSON to back up local data. The app does not auto-migrate existing localStorage data.
